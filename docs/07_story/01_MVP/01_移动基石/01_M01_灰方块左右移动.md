## 用户故事

**As a** 玩家，
**I want** 用键盘 A/D（或方向键、手柄左摇杆）操控一个灰方块在地面左右移动，
**so that** 在接入真实美术与战斗前，先确立"主角 + 输入 + 物理"这条最底层基石，验证手感调参通道（加速度/摩擦力）畅通。

| 编号 | 验收项 | Pass 判据 |

| AC3a | 键盘可移动主角 | 按 A/D 或 ←/→，方块平滑加速左右移动，松手平滑停下 |

## 技术设计要点

### 场景与节点（架构 §3.4 / §3.5）

```
scenes/level.tscn
└── Level (Node2D)
    ├── Ground (StaticBody2D, layer=可碰撞地形(32), mask=0)
    │   └── GroundShape (CollisionShape2D, RectShape2D 960×32)
    └── Player (player.tscn 实例) @ (24, 192)

scenes/player.tscn
└── Player (CharacterBody2D, layer=玩家(1), mask=可碰撞地形(32))  class_name Player
    ├── BodySprite (ColorRect, 青色 32×40 占位, offset 定义尺寸)
    └── BodyShape (CollisionShape2D, RectShape2D 32×40, position=(0,-20))
```

### 脚本契约

- `scripts/player/player.gd`（`class_name Player extends CharacterBody2D`）
  - `@export var stats: PlayerStats`（数据驱动，宪法 §1.4）
  - `_physics_process(delta)`：施重力 → 读水平输入 → 算速度 → `move_and_slide()`
  - `compute_horizontal_velocity(current_vx, input_dir, delta) -> float`：**纯函数**，加速度/摩擦力模型（`move_toward` 逼近目标速度）
- `scripts/resources/player_stats.gd`（`class_name PlayerStats extends StatsResource`）：`move_speed=160 / acceleration=1200 / friction=1200`
- `scripts/resources/stats_resource.gd`（`class_name StatsResource extends Resource`）：基类，含 `max_health=100`

### 关键决策

- **重力走 ProjectSettings**（ADR-2）：`physics/2d/default_gravity=900`，代码用 `get_gravity().y`，不放 PlayerStats。
- **纯函数抽离**（经验沉淀）：`compute_horizontal_velocity` 不依赖 Input/场景树，单测可直接传参，绕开 headless Input 注入难题。
- **占位优先**：用 ColorRect 灰方块，真实精灵留 M3（避免本步因资产配置超时）。
- **碰撞层位掩码**：layer_6 可碰撞地形 = `2^5 = 32`（项目专属经验）；layer_1 玩家 = 1。

### 输入映射（GDD §6）

`project.godot [input]` 段定义 `move_left`/`move_right`，三设备同绑：
- 键盘：A(65)/D(68) + ←/→
- 手柄：左摇杆 axis=0（-1 左 / +1 右）

## 测试策略

### 单元测试（`test/unit/`）

- `test_player_compute_velocity.gd`：
  - 无输入 + 有初速 → 摩擦力逐帧衰减至 0（`assert_almost_eq` 浮点）
  - 满输入 → 加速度逼近 `move_speed`（不超调）

## 验证步骤（§12.5 玩家手工验证）

> 编辑器重载后执行。启动**现有** `level.tscn`，不另建测试关卡。

1. **运行方式**：F5（或 F6 选 level.tscn）启动游戏。
2. **AC3a**：按 A/D 与 ←/→，观察方块平滑左右移动、松手平滑停下。
3. **AC4a**：插手柄，推左摇杆/十字键，观察方块移动。
4. **M1-1**：松开所有键，观察方块稳稳站在地面（不下滑/不穿地）。
5. **M1-2**：在 Inspector 改 `player_stats.tres` 的 `acceleration`/`friction`（如分别调到 400/2400），重新运行，体会起步/刹车的差异。

**预期**：青色灰方块贴地左右移动，起步有加速、松手有滑行，地面稳固。

**异常判定**：方块穿地 → 检查 Ground `collision_layer` 是否 32、Player `collision_mask` 是否 32；按键无反应 → 检查 `project.godot [input]` 是否含 `move_left`/`move_right`。

## 实现备注（已完成）

- 经验沉淀见 `docs/99_postmortem/MEMORY.md`「M1 主角水平移动」条目，要点：
  - 碰撞层是位掩码，layer N → 值 `2^(N-1)`。
  - ColorRect 用 `offset_*` 而非 `size`/`position`。
  - 场景用 MCP 从零构建，禁加载已有场景增量改。
  - GdUnit4 headless 命令用 `bin/GdUnitCmdTool.gd ... --ignoreHeadlessMode`，清理用 `auto_free()`。
