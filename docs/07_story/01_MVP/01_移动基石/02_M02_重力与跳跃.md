## 技术设计要点

### 复用与增量

M1 已有重力下落（`_physics_process` 中 `velocity.y += get_gravity().y * delta`）+ 水平移动。M2 仅增量：

1. `project.godot [input]` 新增 `jump`（Space + 手柄 A）。
2. `player.gd` `_physics_process` 开头加着地判定：
   - `if is_on_floor():` 且 `Input.is_action_just_pressed("jump")` → `velocity.y = stats.jump_velocity`
   - 变速跳跃：`_physics_process` 中 `if Input.is_action_just_released("jump") and velocity.y < 0: velocity.y *= stats.variable_jump_multiplier`

### 数值（PlayerStats，已就绪）

- `jump_velocity = -280.0`（初速度朝上）
- `variable_jump_multiplier = 0.5`（松键上升速度减半）
- 重力 `900`（ProjectSettings，已有）

### 关键决策

- **跳跃判定放主脚本**（架构 §4.3 跳跃判定时机说明铺垫）：M2 仍是散装逻辑，时间戳逻辑（土狼时间/缓冲）留 M6 统一在 Player 主脚本维护。
- **变速跳跃用 `is_action_just_released`**：松键瞬间把上升速度乘倍率，实现可控高度。

## 验证步骤（§12.5）

1. **运行方式**：F5 启动 level.tscn。
2. **AC3b**：按 Space，方块起跳→落下→站稳，不穿地。
3. **M2-1**：按一下立刻松开 Space，再按住 Space，对比两次跳跃高度（松键明显矮）。
4. **M2-2**：起跳后空中按 A/D，观察仍可左右移动。
5. **M2-3**：空中反复按 Space，确认无二段跳。

**预期**：Space 可跳跃，松键变矮，空中可控方向，无双跳。

**异常判定**：按了没跳 → 检查 `jump` input 与 `is_on_floor()`；跳得过高/过低 → 调 `player_stats.tres` 的 `jump_velocity`/`variable_jump_multiplier`。

## 实现备注（已完成）

- 跳跃判定抽纯函数 `should_jump(on_floor, jump_pressed) -> bool`（`return on_floor and jump_pressed`），TDD 小循环覆盖 3 分支（地面+按 / 空中+按 / 地面+没按），见 `test/unit/test_player_jump.gd`。
- `_physics_process` 顺序：重力 → 跳跃判定 → 变速跳跃 → 水平移动 → `move_and_slide()`；变速跳跃用 `Input.is_action_just_released(&"jump") and velocity.y < 0.0` 时 `velocity.y *= variable_jump_multiplier`。
- `project.godot [input]` 新增 `jump`：键盘 Space（physical_keycode=32）+ 手柄 A（InputEventJoypadButton, button_index=0）。
- 经验沉淀见 `docs/99_postmortem/MEMORY.md`「M2 重力与跳跃」条目。
