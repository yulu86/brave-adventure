## 用户故事

**As a** 开发者，
**I want** 把 player.gd 里散装的"移动/跳跃/动画选择"逻辑重构为节点状态机（Idle/Run/Jump），
**so that** 后续战斗态（Attack/Hurt/Dead）和手感逻辑（土狼时间）有清晰的状态容器，避免 if/else 堆砌成不可维护的巨型 `_physics_process`。

> **重构原则**：行为同 M4，不新增功能、不退化手感（架构 §15.1 单一职责 + 纯重构）。

## 技术设计要点

### 新增脚本（架构 §4.2 基类接口）

```
scripts/shared/state.gd           class_name State extends Node
scripts/shared/state_machine.gd   class_name StateMachine extends Node
scripts/player/states/
├── idle_state.gd
├── run_state.gd
└── jump_state.gd
```

### 基类契约（架构 §4.2）

```gdscript
# state.gd
class_name State
extends Node
signal transition_requested(to_state: StringName)   # 备用，主用 machine.transition_to
var machine: StateMachine
var host: Node                  # = StateMachine.get_parent() = Player
func enter(_msg: Dictionary = {}) -> void: pass
func exit() -> void: pass
func process(_delta: float) -> void: pass
func physics_process(_delta: float) -> void: pass
func handle_input(_event: InputEvent) -> void: pass

# state_machine.gd
@export var initial_state: State
var current_state: State
var states: Dictionary[StringName, State] = {}
func transition_to(to_state: StringName, msg: Dictionary = {}) -> void:
    current_state.exit(); current_state = states[to_state]; current_state.enter(msg)
```

> 遵从 ADR-1：节点状态机模式，切换仅改 `current_state` 指针 + 调 enter/exit，**不做 queue_free 重建**；类型化信号（宪法 §5.1）。

### player.tscn 改造

```
Player (CharacterBody2D)
+ StateMachine (Node, initial_state=IdleState)
    ├── IdleState
    ├── RunState
    └── JumpState
```

### 逻辑迁移映射

| 原 player.gd 散装逻辑 | 迁入 |
|---|---|
| 水平输入 + 加速度/摩擦力 | RunState.physics_process（移动时）/ IdleState（静止时 friction 停下） |
| `is_on_floor()` 着地判定 + 跳跃触发 | RunState/IdleState 检测跳跃键 → transition_to("jump") |
| `velocity.y = jump_velocity` | JumpState.enter() |
| 重力施放 | 仍在 Player._physics_process 统一施（或各态 physics_process，二选一，保持一致） |
| 动画选择（pick_animation） | 各 State.enter() 设对应动画 |

### 关键决策

- **重构保护**：先用 M4 的玩家手工验证基线，重构后重跑同一验证（§12.5）确保零退化。
- **host 共享**：State 通过 `host`（Player）读写 velocity/input_dir，状态间数据共享（ADR-1）。

## 验证步骤（§12.5）

1. **运行方式**：F5 启动 level.tscn。
2. **对照 M4 基线**：重复 M3/M4 全部验证步骤，确认移动/跳跃/动画/翻转行为完全一致。
3. **重点回归**：空中方向、变速跳跃、落地回 idle。

**预期**：玩家视角无任何可感知变化（纯重构）。

**异常判定**：任何行为退化（卡帧/跳不起来/动画错）→ 重构引入 bug，立即修复不放过。
