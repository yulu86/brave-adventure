## 用户故事

**As a** 玩家，
**I want** 离开平台边缘的瞬间仍能起跳（土狼时间），以及落地前提前按的跳跃能在落地瞬间自动触发（跳跃缓冲），
**so that** 横版跳跃手感宽容，少踩空、少漏跳，降低挫败感。

---

## 验收标准（AC）

| 编号 | 验收项 | Pass 判据 |
|---|---|---|
| M6-1 | 土狼时间 | 走出平台边缘后 ≤0.1s 内按跳跃，仍能起跳 |
| M6-2 | 跳跃缓冲 | 空中提前按跳跃（落地前 ≤0.1s），落地瞬间自动起跳 |
| M6-3 | 参数可调 | Inspector 改 `coyote_time`/`jump_buffer_time`，宽容度随之变化 |
| M6-4 | 无副作用 | 站立/正常跳跃行为不受影响（不退化 M5） |

## 技术设计要点

### 时间戳维护（架构 §4.3，统一放 Player 主脚本）

```gdscript
# player.gd 新增
var last_grounded_time: float = -INF
var last_jump_press_time: float = -INF

func _physics_process(delta: float) -> void:
    if is_on_floor():
        last_grounded_time = Time.get_ticks_msec() / 1000.0
    if Input.is_action_just_pressed("jump"):
        last_jump_press_time = Time.get_ticks_msec() / 1000.0
    # ... 各 State 通过 host 读写这两个时间戳
```

### 判定逻辑（各地面态 Idle/Run 共用）

```gdscript
# 在 IdleState/RunState.handle_input 或 physics_process
var now: float = Time.get_ticks_msec() / 1000.0
var can_coyote: bool = (now - host.last_grounded_time) <= host.stats.coyote_time
if Input.is_action_just_pressed("jump") and can_coyote:
    machine.transition_to("jump")

# JumpState 落地时
var now: float = Time.get_ticks_msec() / 1000.0
if is_on_floor():
    if (now - host.last_jump_press_time) <= host.stats.jump_buffer_time:
        machine.transition_to("jump")  # 缓冲：落地瞬间再跳
    else:
        machine.transition_to("idle")
```

### 数值（PlayerStats.Jump Feel 组，已就绪）

- `coyote_time = 0.10`（秒）
- `jump_buffer_time = 0.10`（秒）

### 关键决策（ADR-12）

- **时间戳放主脚本**：各地面态通过 `host` 共享，避免每个 State 重复实现、状态不一致。
- **数据驱动**：两参数走 PlayerStats，试玩微调（§12.4/§12.5 用户精调与试玩项）。

## 验证步骤（§12.5）

1. **运行方式**：F5 启动 level.tscn（需有平台边缘可走出，M14 前可用 Ground 边缘）。
2. **M6-1 土狼时间**：走到地面边缘外但刚离开，立即按 Space，确认仍能跳（对照把 `coyote_time` 调 0 后跳不起来）。
3. **M6-2 跳跃缓冲**：空中快落地时提前按 Space，落地瞬间自动起跳。
4. **M6-3 调参**：Inspector 把 `coyote_time` 调到 0.3，感受更宽容；调到 0，确认变严格。
5. **M6-4 回归**：正常站立跳跃不受影响。

**预期**：跳跃手感明显变"宽容"，踩空和漏跳减少。

**异常判定**：始终能跳（土狼时间判定未正确归零）→ 检查 `last_grounded_time` 是否在离地后停止刷新；缓冲无效 → 检查落地分支 buffer 判定。
