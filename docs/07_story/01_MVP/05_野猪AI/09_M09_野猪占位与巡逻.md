## 技术设计要点

### 新增场景与脚本（架构 §3.6 / §4.4）

```
scenes/boar.tscn   class_name Boar (CharacterBody2D)
├── BodySprite (ColorRect 占位)
├── BodyShape (CollisionShape2D, layer=敌人(2), mask=可碰撞地形(32))
├── StateMachine (Node, initial=PatrolState)
│   └── PatrolState
├── Health (Node, stats=boar_stats.tres)
└── 脚本: scripts/enemy/boar.gd
scripts/enemy/states/patrol_state.gd
scripts/resources/boar_stats.gd   class_name BoarStats extends StatsResource
assets/resources/stats/boar_tame.tres（先用 tame 实例）
```

### PatrolState 逻辑（架构 §4.4）

```gdscript
# 在巡逻锚点 [anchor_left, anchor_right] 内移动
func physics_process(delta: float) -> void:
    host.velocity.x = host.facing_dir * host.stats.patrol_speed
    host.move_and_slide()
    if host.global_position.x <= host.anchor_left or host.global_position.x >= host.anchor_right:
        host.facing_dir *= -1   # 到边界翻转
```

### 关键决策

- **复用 M5 状态机基类**（架构 §4.1）：StateMachine/State 不改，直接给 Boar 用，验证基类通用性。
- **巡逻锚点**：在 boar.tscn 用 `@export` 配 `anchor_left`/`anchor_right`（或相对起点 ±range），数据驱动。
- **Boar 与 Player 解耦**（ADR-4）：本里程碑无玩家交互，纯巡逻。

### 可视化搭建协作（§12.4）

Boar 含 ColorRect + CollisionShape2D（可见/碰撞节点），触发 §12.4：AI 搭骨架 + 占位色，用户精调碰撞框尺寸。

## 验证步骤（§12.5）

1. **运行方式**：F5 启动 level.tscn（在 Entities 下放一只 Boar）。
2. **M9-1**：确认方块野猪出现并踩地。
3. **M9-2/3**：观察左右巡逻，速度慢于玩家。
4. **M9-4**：到边界转向，不跑出范围。

**预期**：方块野猪在范围内来回游荡。

**异常判定**：穿地 → layer/mask（敌人 2 / 地形 32）；不转向 → 边界判定逻辑；跑飞 → patrol_speed/锚点配置。
