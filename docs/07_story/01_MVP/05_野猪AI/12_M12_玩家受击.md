> 玩家 Hurtbox layer_5 = 16，被敌人攻击区 layer_4 = 8 检测。

### 伤害链（野猪 Hitbox → 玩家 Hurtbox → 玩家 Health）

复用 M8 的 hitbox/hurtbox/health 组件（ADR-10 场景化复用）：
- 野猪 Hitbox（layer=敌人攻击区(8), mask=受击区(16)）在 Chase 态 monitoring=true（M11 已开）
- 接触玩家 Hurtbox → 玩家 Health.take_damage(boar.contact_damage=20)
- Health.health_changed → Player 监听 → transition_to("hurt")（或 dead）

### HurtState 逻辑（架构 §4.3 + ADR-3 击退归状态 + ADR-6 无 hurt 动画）

```gdscript
var _invincible_timer: float = 0.0
func enter(_msg := {}) -> void:
    host.body_sprite.play("idle")   # ADR-6：无 hurt 动画，用 idle 闪烁代替
    # 击退（ADR-3）：施加远离伤害源的速度
    host.velocity = host.knockback_dir * host.stats.knockback
    _invincible_timer = host.stats.invincible_duration
func physics_process(delta: float) -> void:
    # 击退速度衰减
    host.velocity = host.velocity.move_toward(Vector2.ZERO, host.stats.friction * delta)
    # 闪烁（modulate.a 拉锯）
    host.body_sprite.modulate.a = 0.5 if fmod(_invincible_timer, 0.1) < 0.05 else 1.0
    _invincible_timer -= delta
    if _invincible_timer <= 0:
        host.body_sprite.modulate.a = 1.0
        machine.transition_to("idle")
```

### 无敌帧判定

- HurtState 期间，Health.take_damage 因 `is_hurt_state` 或无敌计时被忽略（Health 加 `_invincible` 标志或 Player 在 HurtState 拦截）。
- 简化：HurtState 进入时设 `host.is_invincible = true`，exit 时 false；Hitbox 命中前检查。

### 关键决策

- **击退归状态**（ADR-3）：不在 Health 组件处理。
- **无 hurt 动画**（ADR-6）：idle 帧 + modulate.a 闪烁临时代替。
- **Attack→Hurt**（架构 §14.2）：攻击中受击可中断，状态机图已含。

### 可视化搭建协作（§12.4）

野猪 Hitbox 尺寸/偏移、玩家 Hurtbox 尺寸属精调项。

## 验证步骤（§12.5）

1. **运行方式**：F5 启动 level.tscn。
2. **AC6b**：引野猪撞到玩家，确认血量减少（M19 前 HUD 未就绪，可临时 print 或看调试）。
3. **M12-1**：观察受击瞬间被弹开。
4. **M12-2**：观察闪烁期间再被撞不扣血。
5. **M12-3**：无敌结束确认回 Idle。
6. **M12-4**：攻击中被撞，确认切 Hurt。

**预期**：被撞扣血、弹开、闪烁无敌、无敌结束回 idle、攻击可被打断。

**异常判定**：连续扣血 → 无敌帧未生效；不击退 → knockback 方向/衰减；攻击中被打不进 Hurt → Attack→Hurt 切换缺失。
