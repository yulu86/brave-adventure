## 技术设计要点

### 关卡设计增量

在 GroundLayer/GeometryLayer 用瓦片绘制 2~3 个高低差平台（段②密林区域），高度 2~3 格（每格 16px）。

```
         ┌──────┐  平台3 (48px高)
         │      │
    ┌────┘      │  平台2 (32px高)
    │           │
════╧═══════════╧══  地面
```

### 跳跃可达性校验

- jump_velocity = -280，gravity = 900
- 最大跳跃高度 ≈ `v²/(2g)` = `280²/1800` ≈ 43px ≈ 2.7 格
- 平台高度 ≤ 2 格（32px）必达，3 格（48px）需贴近起跳（验证 variable jump 上限）

### 关键决策

- **平台用 TileMapLayer 绘制**（复用 GroundLayer/GeometryLayer，collision_layer=可碰撞地形(32)）。
- **手感验证关卡**：本里程碑是 M6 手感的实战检验场，若跳不上 3 格平台，回调 PlayerStats.jump_velocity。

### 可视化搭建协作（§12.4 强制关卡）

平台瓦片绘制是用户精调项，触发 §12.4：AI 给平台坐标/高度建议，用户在编辑器绘制瓦片。

## 验证步骤（§12.5）

1. **运行方式**：F5 启动 level.tscn。
2. **M15-1**：确认平台存在（2~3 格高）。
3. **M15-2**：尝试跳上各平台，确认可达。
4. **M15-3**：跳上后站立稳固。
5. **M15-4**：用边缘起跳/落地接跳验证 M6 手感。

**预期**：能跳上 2~3 格平台，手感顺滑。

**异常判定**：跳不上 3 格 → 调 jump_velocity（或降平台高度）；站不住滑落 → 平台瓦片碰撞面配置。
