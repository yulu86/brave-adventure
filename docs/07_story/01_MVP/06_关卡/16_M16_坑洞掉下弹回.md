## 用户故事

**As a** 玩家，
**I want** 掉进坑洞时扣 1 颗心血量并自动弹回最近平台，而不是卡死或直接死亡，
**so that** 失误有惩罚（扣血）但不会卡关，保持友好试错体验。

## 技术设计要点

### 坑洞检测区

在坑洞下方放一个 Area2D 检测玩家掉落：

```
PitDetector (Area2D, layer=触发区(64), mask=玩家(1))
└── CollisionShape2D (RectShape2D 覆盖坑洞区域)
```

> 触发区 layer_7 = 64，玩家 layer_1 = 1（与 MineEntrance 同层语义）。

### 弹回逻辑（level_controller.gd 或 player.gd）

```gdscript
func _on_pit_body_entered(body: Node2D) -> void:
    if body.is_in_group("player"):
        body.health.take_damage(pit_damage)   # 扣 20
        body.global_position = nearest_platform_pos   # 弹回
        body.velocity = Vector2.ZERO
        # 触发短暂无敌（复用 HurtState 无敌机制或临时标志）
```

### 数值

- `pit_damage = 20`（1 颗心，GDD §3.2）
- 弹回位置 = 坑洞两侧最近平台坐标（关卡配置 `@export var respawn_points`）

### 关键决策

- **坑洞走触发区层**（layer_7=64）：与伤害链隔离，纯交互判定。
- **弹回而非死亡**：GDD §3.2 设计，友好试错。
- **复用 Health.take_damage**：扣血走统一链路（HUD 在 M19 联动）。
- **无敌保护**：弹回瞬间设无敌，避免边缘反复触发。

### 可视化搭建协作（§12.4）

PitDetector 区域位置/尺寸、respawn_points 坐标属用户精调项。

## 验证步骤（§12.5）

1. **运行方式**：F5 启动 level.tscn（含坑洞）。
2. **AC9**：主动跳进坑洞，确认扣血 + 弹回。
3. **M16-1**：弹回后确认能正常移动，不卡死。
4. **M16-2**：确认弹回到最近平台（非起点）。
5. **M16-3**：弹回瞬间反复进出，确认无敌保护不连续扣血。

**预期**：掉坑扣 1 心弹回，不卡死不连续扣血。

**异常判定**：卡坑底 → 弹回逻辑/位置未生效；连续扣血 → 无敌标志缺失；不扣血 → take_damage 未调或 group 未配。
