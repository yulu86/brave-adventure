## 技术设计要点

### 新增场景与脚本（架构 §3.2 / ADR-8）

```
scenes/hud.tscn   class_name HUD (CanvasLayer, layer=10)
├── MarginContainer (anchors=左上)
│   └── HeartsBox (HBoxContainer)
│       └── Heart[0..4] (TextureRect × 5, AtlasTexture 亮/暗态)
└── 脚本: scripts/ui/hud.gd
```

### hud.gd 契约（架构 §3.2）

```gdscript
class_name HUD
extends CanvasLayer

@onready var hearts: Array[TextureRect] = [$MarginContainer/HeartsBox.get_children()]

## 按 hp 切换心的亮/暗（每 20 血 1 颗，被动接收，不自改血量）
func update_health(hp: int) -> void:
    var lit_count: int = ceili(float(hp) / 20.0)
    for i in range(hearts.size()):
        hearts[i].texture = bright_atlas if i < lit_count else dim_atlas
```

### GameManager 挂载（架构 §3.1）

```
GameManager
+ HUD (hud.tscn 实例)   # 子场景根自带 CanvasLayer(layer=10)
```

```gdscript
# game_manager.gd
@onready var hud: HUD = $HUD
func _ready():
    ...
    # 连接 Player 血量信号（LevelController 转发或直连）
func update_health(hp: int) -> void:
    hud.update_health(hp)
```

### 信号链（架构 §7.1）

```
Player.Health.health_changed(hp) → LevelController 转发 → GameManager.update_health(hp) → HUD.update_health(hp)
```

### 资产（GDD §7.2）

- 心用 `HUD/Base-01.png` 的生命球切片，`AtlasTexture` 切亮/暗两态。
- 每颗 = 20 HP，5 颗 = 100 HP（GDD §4.2）。

### 关键决策

- **HUD 独立自包含子场景**（ADR-8）：根自带 CanvasLayer(layer=10)，可脱离 GameManager 独立测试。
- **HUD 被动接收**（架构 §2.1）：不自改血量，只接收 update_health 刷新。
- **信号向上**（宪法 §1.2）：Player 发 health_changed → LevelController → GameManager → HUD。

### 可视化搭建协作（§12.4）

Heart 的 AtlasTexture 切片区域、MarginContainer 间距属精调项。

## 验证步骤（§12.5）

1. **运行方式**：F5 启动 level.tscn。
2. **AC5**：确认左上角 5 颗心满血全亮。
3. **M19-1/2**：引野猪撞/掉坑（M16），观察心随血量变暗。
4. **M19-3**：切场景（若已实现）确认 HUD 常驻。

**预期**：5 颗心，受伤联动变暗。

**异常判定**：心不变 → 信号链断（health_changed 是否连到 HUD）；数量错 → lit_count 计算；位置错 → MarginContainer anchors。
