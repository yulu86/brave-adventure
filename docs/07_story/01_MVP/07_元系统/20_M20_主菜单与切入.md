## 用户故事

**As a** 玩家，
**I want** 启动游戏先看到带森林背景的标题主菜单，点"开始冒险"后画面无缝淡出进入关卡，
**so that** 有完整的游戏入口体验，且主菜单→关卡的过渡丝滑（BGM 交叉淡出、画面不跳）。

## 技术设计要点

### 新增场景与脚本（GDD §8.7）

```
scenes/main_menu.tscn
├── ParallaxBackground (Background.png)        ← 复用 level 视差
├── TileMapLayer: Background/Ground            ← 复用，一段草地
├── Tree 装饰 (Green-Tree.png × 2~3)           ← 复用
├── Camera2D (静态, 位置=关卡起点取景)
└── MenuUI (CanvasLayer layer=100, process_mode=ALWAYS)
    ├── TitleLabel (勇士的冒险 / BRAVE ADVENTURE)
    ├── ButtonGroup (VBoxContainer: 开始冒险 / 退出游戏)
    └── VersionLabel
scripts/ui/main_menu.gd
```

### main_menu.gd 契约

```gdscript
class_name MainMenu
extends Node2D

signal start_requested   # 向上发，由场景流处理（或直接 change_scene）

func _ready() -> void:
    $MenuUI/ButtonGroup/StartButton.grab_focus()   # 入口聚焦
func _on_start_pressed() -> void:
    # MenuUI 淡出 + BGM 交叉淡出 → 切关卡
    var tween := create_tween()
    tween.tween_property($MenuUI, "modulate:a", 0.0, 0.3)
    tween.tween_callback(_change_to_level)
func _change_to_level() -> void:
    GameManager.crossfade_bgm(forest_stream, 0.3)
    get_tree().change_scene_to_file("res://scenes/level.tscn")
```

### BGM 交叉淡出（GDD §8.6）

GameManager 新增 `crossfade_bgm`：主菜单 `time_for_adventure` 淡出 ↔ 关卡 `forest` 淡入（各 0.3s）。

### 启动场景

`project.godot [application] run/main_scene = res://scenes/main_menu.tscn`（替换原直接进 level）。

### 关键决策

- **主菜单背景与关卡一致**（GDD §8.7）：取景坐标/zoom 与关卡起点 Camera2D 一致，切换无跳变。
- **MenuUI 淡出**（GDD §8.7）：Tween modulate.a → 0，0.3s。
- **按钮可访问性**（宪法 §8）：grab_focus + focus 链。
- **信号向上**（宪法 §1.2）：按钮点击向上发，场景切换由主菜单或 GameManager 处理。

### 可视化搭建协作（§12.4 强制关卡）

主菜单含 Camera2D/Control/按钮等可见节点，触发 §12.4：AI 搭骨架 + 默认布局，用户精调标题位置/按钮样式/取景。

## 验证步骤（§12.5）

1. **运行方式**：F5 启动（现从 main_menu 起）。
2. **AC1**：确认见主菜单（背景+标题+按钮）。
3. **M20-1**：键盘 Tab/手柄确认按钮可聚焦，开始按钮默认聚焦。
4. **AC2**：点"开始冒险"，观察 MenuUI 淡出 + BGM 交叉淡出 + 进关卡，确认画面无跳变。
5. **M20-2**：对比主菜单与关卡段①背景一致性。

**预期**：标题画面→点开始→无缝进关卡。

**异常判定**：画面跳变 → 取景坐标/zoom 不一致；BGM 硬切 → crossfade 未生效；按钮不可聚焦 → grab_focus/focus 链。
