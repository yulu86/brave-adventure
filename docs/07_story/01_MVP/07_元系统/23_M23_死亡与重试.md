## 用户故事

**As a** 玩家，
**I want** 血量归零时进入死亡画面，点"重试"重新加载关卡、状态重置，
**so that** 失败有明确反馈且能快速重试，不会因死亡卡死。

## 技术设计要点

### Player DeadState（架构 §4.3，M12 已搭骨架）

```gdscript
# dead_state.gd（补全）
func enter(_msg := {}) -> void:
    host.body_sprite.play("dead")
    host.body_sprite.animation_finished.connect(_on_finished)
    host.hitbox.monitoring = false
    host.hurtbox.monitoring = false
func _on_finished() -> void:
    host.health.died.emit()   # 播完发 died → LevelController → GameManager 切 death_screen
```

### 信号链（架构 §7.1）

```
Player.Health.died → LevelController._on_player_died → GameManager.on_player_died
  → BGM 淡出 + HUD 隐藏 → change_scene_to_file(death_screen.tscn)
```

### 重试逻辑（架构 §8.1 全局血量态重置）

```gdscript
# death_screen.gd
func _on_retry_pressed() -> void:
    GameManager.reset_player_state()   # 全局血量重置
    get_tree().change_scene_to_file("res://scenes/level.tscn")

# game_manager.gd
func reset_player_state() -> void:
    global_player_hp = 100   # 供重玩时重置 Player 血量
    hud.update_health(100)
    play_bgm(forest_stream)   # BGM 重新淡入
```

### 关键决策

- **DeadState 播 dead 动画后发 died**（架构 §4.3）：非立即切场景，给死亡动画时间。
- **重试重置全局血量态**（架构 §8.1/§14.2）：GameManager 持全局血量，重玩主动 reset，避免跨场景同步问题。
- **BGM 重试淡入**（GDD §8.6）：重试时 forest 重新淡入。
- **死亡 shader 留 M25**：本里程碑先做 UI/流程，碎裂震动 shader 后接。

## 验证步骤（§12.5）

1. **运行方式**：F5 进关卡，主动送死（被野猪撞 5 次或反复掉坑）。
2. **AC11/M23-1**：血量归零，确认播 dead 动画后切死亡画面。
3. **AC12/M23-2**：点"重试"，确认关卡重载、血量满、野猪/位置重置。
4. **M23-3**：点"返回标题"，确认切回主菜单。
5. **BGM**：死亡 BGM 淡出，重试 BGM 淡入。

**预期**：血量归零→死亡画面→重试重玩（状态重置）/返回标题。

**异常判定**：不切死亡画面 → died 信号链；重试血量没重置 → reset_player_state；卡死 → reload 异常。
