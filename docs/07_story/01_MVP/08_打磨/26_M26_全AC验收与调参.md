## 技术设计要点

本里程碑**无新功能开发**，是端到端验收 + 调参阶段：

1. **逐条 AC 核验**：按上表逐条操作验证，记录 Pass/Fail。
2. **失败修复循环**：任一 AC Fail → 定位里程碑 → 最小改动修复 → 重跑诊断/lint → 重载编辑器 → 再验证（§12.5 循环）。
3. **调参项**（§12.4/§12.5 用户精调与试玩）：
   - 手感：`coyote_time`/`jump_buffer_time`/`jump_velocity`/`acceleration`/`friction`
   - 战斗：`attack_cooldown`/`invincible_duration`/`knockback`
   - 难度：三段 `chase_speed`/`detect_range`
   - 音频：BGM 音量基线、暂停降至比例
   - Shader：胜负效果强度
4. **端到端功能测试**（`test/functional/`）：关卡通关流、死亡重试流脚本化输入 + 截图。

### 端到端测试脚本（架构 §11）

- `test/functional/test_full_playthrough.gd`：主菜单→关卡→战斗→终点→胜利 全流程。
- `test/functional/test_death_retry.gd`：送死→死亡画面→重试→重玩。
- `test/functional/screenshots/`：关键节点截图存证。

### 手柄端到端验收（AC4）

M1 配 input action 时已绑多设备，M26 做完整手柄试玩：左摇杆移动、A 跳、X 攻击、Start 暂停。

### 关键决策

- **玩家驱动验收**（宪法 §12.5）：AI 不得擅自宣布通过，由玩家逐条核验。
- **设计-实现一致性**（宪法 §13.3）：对照架构文档与实际代码/场景，偏差即修正。
- **全门禁达标**（宪法 §10）：G01~G13 全部通过后方可交付。

## 验证步骤（§12.5）— 完整端到端

> 启动**现有**游戏（main_menu 起），按 AC1~AC14 顺序逐条操作核验。

1. **AC1**：启动见主菜单。
2. **AC2**：点开始，观察无缝切入。
3. **AC3**：键盘移动/跳/攻。
4. **AC4**：插手柄重复 AC3 + Start 暂停。
5. **AC5**：受击观察心。
6. **AC6**：靠近野猪观察 AI。
7. **AC7**：挥剑 2 击杀野猪。
8. **AC8**：对比三段野猪速度。
9. **AC9**：掉坑观察扣血弹回。
10. **AC10**：走到终点观察胜利。
11. **AC11**：送死观察死亡。
12. **AC12**：点重试观察重玩。
13. **AC13**：Esc 暂停三按钮。
14. **AC14**：全程听 BGM 连续性。

**调参**：根据试玩感受微调上述参数项，迭代至手感/难度/音画满意。

**预期**：全 14 条 AC Pass，手感/难度/音画达可交付质量。

**异常判定**：任一 AC Fail → 进入修复循环（最小改动→诊断→重载→再验证），由玩家驱动直到全 Pass。
